# Wasla Production Hardening Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring Wasla to production-ready state for team Resal — fix all P0/P1/P2 issues from the 2026-05-05 audit, harden every page across mobile (320–767 px), tablet (768–1023 px), and desktop (1024 px+), verify the print/PDF report path, and tighten theming/a11y/perf.

**Architecture:** Five independent workstreams executed by parallel subagents:
- **Wave A — Performance:** route-level code splitting, manualChunks, drop unused deps (moment, lodash), update browserslist
- **Wave B — Theming:** introduce semantic status tokens, replace hard-coded reds/blues across pages, fix App.jsx loading spinner
- **Wave C — Accessibility:** heading hierarchy, inline form validation (aria-invalid + aria-describedby), 44 px touch targets
- **Wave D — Visual polish:** rework MetricCard, fix Card-with-built-in-shadow conflict, drop the dead `box-shadow: none` global, typography pairing for headings
- **Wave E — Print/Production verification:** verify InspectionReport print, test responsive matrix, run typecheck + build, smoke test

Waves A–D run in parallel. Wave E runs after, sequentially, in the main session.

**Tech Stack:** React 18, Vite 6, Tailwind 3, Radix UI, Supabase, Recharts, Framer Motion, jsPDF, html2canvas, sonner.

**Branch:** `main` (per user direction).

Write `src/lib/AppLoader.jsx`:

```jsx
import React from "react";

export default function AppLoader({ label = "Loading…" }) {
  return (
    <div
      role="status"
      aria-live="polite"
      aria-label={label}
      className="fixed inset-0 flex items-center justify-center bg-background"
    >
      <div className="w-8 h-8 border-4 border-muted border-t-primary rounded-full animate-spin" />
      <span className="sr-only">{label}</span>
    </div>
  );
}
```
