## Design Context

### Users
Wasla serves two distinct user groups with different contexts:
- **Field inspectors** on mobile devices — conducting on-site property walkthroughs, capturing photos, filling inspection reports. They work in variable lighting, often one-handed, and need fast, reliable interactions with minimal friction.
- **Office managers** on desktop — reviewing completed inspections, managing client relationships, generating invoices, and monitoring business metrics from dashboards.

The job to be done: streamline property inspection workflows from field capture to client billing, replacing paper forms and fragmented tools with a single platform.

### Brand Personality
**Modern, Efficient, Bold** — Wasla positions itself as a professional-grade tool that feels premium without being cold. It should communicate competence and authority while remaining approachable. Think "trusted business partner" not "enterprise software."

### Aesthetic Direction
- **Clean & functional** — data-first, decoration-never philosophy. Refine what exists rather than reinvent.
- **Neutral slate palette** with emerald green (`#059669`) reserved for the PWA theme and key brand moments, not saturating the UI.
- **HSL CSS variable system** (shadcn/ui) for light and dark mode. Dark mode uses deep grey-blue (`220 16% 5%`) for reduced eye strain.
- **No box shadows** — separation through subtle background shifts and borders.
- **System fonts** — no custom web fonts loaded; rely on the native font stack for speed.
- **Radix UI primitives** (shadcn/ui) as the component foundation — consistent, accessible, composable.
- **Framer Motion** for meaningful transitions, not decorative animation.
- **Border radius**: `0.375rem` (6px) — functional, not bubbly.

### Design Principles

1. **Data first, decoration never** — Every pixel serves the user's task. No ornamental gradients, glows, or glassmorphism. If an element doesn't help the user read, decide, or act, remove it.

2. **Field-ready by default** — Design for the inspector holding a phone in bright sunlight with one hand. Touch targets >= 44px, font sizes >= 16px on mobile, high contrast, forgiving tap zones. Desktop refinements are additive, not primary.

3. **Calm confidence** — The UI should feel like a quiet, competent tool. Neutral tones dominate. Color enters only to communicate meaning (status, alerts, actions). Emerald green marks primary actions and brand identity sparingly.

4. **Two-context harmony** — Mobile and desktop are different tools for different jobs, not scaled versions of each other. Mobile prioritizes capture and quick actions; desktop prioritizes review, analysis, and management. Shared components adapt, not just shrink.

5. **Accessible by instinct** — WCAG AA contrast as the floor. Touch targets sized for field conditions. Reduced motion respected. Inputs that don't trigger iOS zoom. Safe-area awareness for notched devices. Accessibility isn't a checklist — it's how the tool earns trust.
